# AgentHub Visual Manager

AgentHub Visual Manager 是一个中文优先、本地运行的 Agent 协作观察与受控开发控制台。本仓库是 **v0.2.0 的公开归档快照**，已停止主动维护。

## 能力

- 只读观察 `.agent-hub` 项目状态、任务、运行记录与评审证据。
- 在浏览器内导入本地 `.agent-hub` 目录；导入内容不上传、不回写、不持久化。
- 为可信的本地 Git 工作树提供受限的顺序式多角色开发流程。
- 只允许经过校验的文本补丁、仓库声明的固定验证命令和 localhost 浏览器验收。
- Provider Key 仅保留在当前页面内存中；产品不提供 Stage、Commit 或 Push 操作。
- 归档版默认禁用 checkpoint 持久化与跨重启恢复。显式使用 `--enable-checkpoints` 时只保存脱敏后的结构进度：原始任务、阶段叙述、普通回执正文和失败详情会在落盘前移除，恢复后不会还原这些正文。

这不是多个 CLI 编码进程并行组成的 Agent 舰队，也不是远程构建或生产发布平台。

## 快速开始

要求：Windows、Node.js `^20.19.0` 或 `>=22.12.0`，以及 npm 10 或更高版本。

最简单的方式是双击：

```text
start-all.bat
```

启动器会在需要时安装依赖、构建前端，并在 `http://localhost:8794` 打开本地页面。停止服务请双击：

```text
stop-all.bat
```

开发模式：

```bash
npm ci --ignore-scripts
npm run dev
```

`npm run dev` 仅用于前端与合成数据预览；完整本地能力请使用 `start-all.bat` 的同源页面。依赖安装会访问 npm registry。

验证：

```bash
npm run build
npm test
```

- 仅对你信任的本地仓库使用“独立开发”；受管命令仍以当前操作系统用户权限执行。

- `.agent-hub` 是无真实数据的演示工作区。需要观察其他项目时，请在本地显式选择对应目录。
- 本项目已经归档，不承诺安全更新、兼容性维护或问题响应。继续使用前请自行审计。

- 仓库内置的任务编号、日期、提交哈希和路径均为合成占位数据，不对应原开发环境。
- 原控制仓名称、本地目录、历史 HEAD、审批回执和接管提示均未包含在本快照中。
- 因未随项目保留可复核的再分发许可链，归档版不包含生成式 3D 角色模型或独立舞台图片；界面使用代码生成的轻量角色与背景。
